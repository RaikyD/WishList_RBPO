P09 — SBOM, SCA и Evidence

Этот каталог содержит артефакты, автоматически генерируемые security-workflow
.github/workflows/security-p09.yml в рамках практики P09.

Структура каталога:
Файл	Описание
sbom.json	Последний SBOM проекта, сгенерированный Syft (фиксированная версия — v1.18.0).
sbom-<commit>.json	SBOM, привязанный к конкретному коммиту (GITHUB_SHA).
sca_report.json	Полный отчёт Grype об уязвимостях на основе SBOM.
sca_summary.md	Человекочитаемая текстовая сводка по severity, с примерами и планами действий.

SBOM/SCA формируются автоматически как артефакты GitHub Actions
и синхронизируются в этот каталог в рамках CI.

Как это работает:

При push или ручном запуске (workflow_dispatch) workflow:

генерирует SBOM через Syft,

анализирует SBOM через Grype,

формирует отчёт (sca_report.json) и краткую сводку (sca_summary.md),

выкладывает артефакты через upload-artifact.

Все проверки выполняются строго на фиксированных версиях инструментов,
что обеспечивает воспроизводимость.

Артефакты привязаны к коммитам — SBOM и отчёт можно сопоставить
с конкретной версией кода.

Использование артефактов
Артефакты применяются:

при анализе рисков (раздел DS итогового отчёта);

при принятии решений о фиксе или waiver'ах уязвимостей;

для сравнения динамики SCA между коммитами;

при аудите безопасности.

Связь с политикой уязвимостей

Базовая политика и waivers находятся в policy/waivers.yml.

Policy определяет:

как обрабатываются Critical/High/Medium/Low уязвимости,

обязательность фикса/waiver'а,

сроки пересмотра waiver’ов.
